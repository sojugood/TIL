Classes 상속

상속 : 기존 클래스의 속성과 메서드를 물려받아 새로운 하위 클래스를 생성하는 것

상속이 필요한 이유
 1. 코드 재사용
  - 상속을 통해 기존 클래스의 속성과 메서드를 재사용할 수 있음
  - 새로운 클래스를 작성할 때 기존 클래스의 기능을 그대로 활용할 수 있으며, 중복된 코드를 줄일 수 있음

 2. 계층 구조
  - 상속을 통해 클래스들 간의 계층 구조를 형성할 수 있음
  - 부모 클래스와 자식 클래스 간의 관계를 표현하고, 더 구체적인 클래스를 만들 수 있음

 3. 유지 보수의 용이성
  - 상속을 통해 기존 클래스의 수정이 필요한 경우, 해당 클래스만 수정하며 되므로 유지 보수가 용이해짐
  - 코드의 일관성을 유지하고, 수정이 필요한 범위를 최소화할 수 있음


다중 상속 시 class abc(A, B): 일 경우 , A와 B가 동일한 클래스 변수를 가지고 있을 때, 먼저 상속 선언한 A의 클래스 변수를 할당함 / 메서드도 동일 -> 이는 메서드 결정 순서(MRO)에 의한 것

메서드 결정 순서(MRO) : 다중 상속을 지원하는 언어에서 발생하는 다이아몬드 상속 등의 상속 구조에서 어떤 메서드가 호출되어야 하는지를 결정하는 규칙. 파이썬에서는 C3 선형화 알고리즘이 MRO를 결정하는 방식으로 사용
 인스턴스 -> 자식 클래스 -> 부모 클래스
